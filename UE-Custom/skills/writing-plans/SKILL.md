# 编写实施计划（UE 适配）

## 概述
输出清晰、可执行的计划，面向不熟悉上下文的工程师。测试与验证为建议项，不强制要求执行或回填结果。

**开头声明**："我正在使用 writing-plans 技能编写实施计划。"

**计划保存路径（建议）**：由项目约定路径决定

## 任务粒度
每一步控制在 2-5 分钟：
- 明确修改点
- 最小实现
- 建议验证步骤（可选）
- 必要的记录

## 计划头部模板
```markdown
# [功能名称] 实施计划

**目标**：一句话说明要实现的内容

**架构**：2-3 句说明整体思路

**技术点**：UE 模块/系统/关键类

---
```

## 任务模板（示例）
```markdown
### 任务 N：[模块/组件]

**文件**：
- 新增：`路径`
- 修改：`路径`
- 配置：`路径`

**步骤 1：准备验证（可选）**
- 有 AutomationTest：写出最小失败场景
- 无自动化测试：写出 PIE/Editor 复现步骤与预期结果

**步骤 2：最小实现**
- 写出最小改动（可给出关键代码块）

**步骤 3：建议验证（可选，真人执行 PIE）**
- 写出 PIE/Editor 验证步骤与证据建议

**步骤 4：记录（可选）**
- 若有验证，回填 `./UE验证证据模板.md`
```

## 提示
- 文件路径要具体
- 配置改动写明键名与回滚方式
- 调试插桩建议使用 `UE_LOG` / `DrawDebug*`
- 不强制执行测试或回填 PIE 结果
