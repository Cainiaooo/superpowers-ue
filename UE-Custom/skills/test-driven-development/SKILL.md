# 验证优先（UE 适配）

## 概述
UE 项目中自动化测试不总是可用。本技能将“测试优先”调整为“验证优先（建议）”，在实现前明确验证方式即可。验证不是强制门槛，但建议尽量提供可复现的步骤与证据。

**PIE 验证建议由真人开发者执行，AI 不代跑，仅提供步骤与证据建议。**

## 何时使用
- 新功能
- Bug 修复
- 行为变更
- 重构

## 建议流程（可选执行）
1. **定义验证目标**
   - 有 AutomationTest：建议写出最小失败场景
   - 无自动化测试：写出 PIE/Editor 步骤、预期结果、日志关键行
2. **最小实现**
   - 只做满足目标的最小改动
3. **可选验证**
   - 真人执行 PIE/Editor 或运行测试
   - 记录日志/截图/录屏（可选）
4. **可选记录证据**
   - 参考 `./UE验证证据模板.md`

## 注意事项
- 若本次未验证，建议说明原因与后续计划。
- 调试插桩建议使用 `UE_LOG` / `DrawDebug*`，完成后可降级或清理。
- 避免过度设计，遵循最小改动。
