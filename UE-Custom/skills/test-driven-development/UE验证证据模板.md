# UE 验证证据模板

## 目的
本模板用于记录 UE 项目功能变更后的验证证据。PIE/Editor 验证建议由真人开发者执行，AI 仅给出步骤与证据建议。

## 填写说明
- 变更完成后建议填写“验证类型 + 证据”，不强制要求。
- 若无 AutomationTest，可选择提供 PIE/Editor 或日志证据。
- 证据建议可追溯（日志关键行、截图/录屏路径、复现步骤）。

## 模板（复制后填写）
```
【变更/任务】
- 标题：
- 关联文件：
- 关联模块：

【验证类型】
- AutomationTest / PIE / Editor / Log-Only

【PIE/Editor 验证（真人执行，建议填写）】
- 执行者：
- 日期：
- 引擎版本：
- 平台/配置：
- 地图/关卡：
- GameMode：
- 角色/控制器：
- 输入操作：
- 复现步骤：
  1)
  2)
- 期望结果：
- 实际结果：
- 证据：
  - 日志关键行：
  - 截图/录屏路径：

【AutomationTest（若适用）】
- 测试命令：
- 关键输出：
- 失败 -> 通过 说明：

【Log-Only（若适用）】
- 日志分类：
- 关键字段：
- 触发步骤：
- 日志片段：
```

## 备注
- 若由 AI 提供验证步骤，可在“PIE/Editor 验证”中回填真人执行结果（可选）。
- 若未进行验证，可说明原因与后续计划（可选）。
